# 算子与计算图
算子是计算图的节点

每个算子对应计算图中的一个节点
算子的输入输出对应节点的边
算子的属性作为节点的属性
算子执行顺序由计算图决定

计算图的拓扑序决定了算子的执行顺序
有依赖关系的算子必须按序执行
算子内存分配

算子的输入输出tensor内存由计算图统一管理
可以复用中间结果的内存空间
算子调度

计算图负责调度算子在不同设备上执行
可以实现算子级别的并行
支持算子间的流水线执行
算子性能分析

全局分析计算图中算子的执行情况

# Tensor和buffer
tensor和buffer都是数据类型,但是语义不同,作用的层次不同
tensor是模型的权重,算子的输入输出,计算图的边
Tensor类的核心设计考虑
    能够完整描述多维张量
    唯一标识 - 每个 Tensor 有唯一名称，模型内部不允许重名
    简单易用 - 丰富的构造函数，支持外部数据传入、内存池分配、设备直接分配等多种创建方式
    数据传输 - 支持跨设备数据传输和设备间数据拷贝
    内存管理 - 通过引用计数方式实现内部数据自动管理
    动态形状 - 支持动态 Shape

Buffer的核心设计考虑
    能够完整的描述内存
    设备关联 - 与分配内存的设备关联
    服务Tensor
    数据传输
    内存管理
    序列化
