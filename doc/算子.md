# 算子优化的挑战

• 优化手段多样：要在不同情况下权衡优化及其对应参数，对于优化专家来说也是相当耗费精力
• 通用性与移植性：不同类型的硬件架构差异，使得优化方法要考虑的因素也有很大不同
• 优化间相互影响：各种优化之间可能会相互制约，相互影响。这意味着找到最优的优化方法组合与序列就是一个困难的组合优化问题，甚至是 NP 问题。

## 调度优化
调度树
Root node
树的入口，表示整个调度的顶层上下文。

Loop node
表示某一维的遍历顺序（例如先 y 后 x）。
也携带执行属性：串行 / 并行 / 向量化。
你可以把它看成一层 for 循环壳。

Storage node
表示是否把中间结果“落地存储”（buffer/cache）。
有 storage，通常意味着“先算出来存起来，后面复用”。

Compute node
真正执行计算的节点（叶子最常见）。
也可以挂子 compute 节点，表示“内联计算”而不是先写中间 buffer 再读。


循环展开（Loop Unrolling） 降低分支与循环控制开销 提高 ILP（指令级并行）机会 有利于后续向量化 代码体积变大 可能增加寄存器压力 展开过度时收益会下降
循环分块（loop tiling）一维 tiling：常用于 1D 数据（如向量、序列）。二维/三维 tiling：常用于矩阵/张量，更能提升 cache 局部性与数据复用。
循环重排（loop Reorder） 就是交换循环嵌套顺序，不改计算结果（在依赖允许时），只改变访问顺序和局部性。
循环融合（loop Fusion）
循环拆分（loop Split）
向量化（Vector）
张量化（Tensor）
访存延迟（Latency Hiding）  常见做法是预取、双缓冲、流水化、增加并行度、计算与数据搬运重叠（overlap），把等待内存的时间藏在别的指令执行里。
CPU：
• 延迟隐藏可以通过多线程，或者硬件隐式数据预取实现
GPU：
• 依赖于 Wrap Schedule 对多线程的管理调度和上下文切换实现
NPU/TPU：
• 采用解耦访问/执行（Decoupled Access/Execute，DAE）架构
理解（简要）：
核心思想都一样：内存慢，就让计算别空等。
CPU：靠 SMT/多线程 + 预取器提前拉数据。
GPU：靠 warp（图里写成 Wrap，应为 Warp）快速切换，别的 warp 顶上。
NPU/TPU：把“搬数据”和“做计算”拆成两条流水并行推进（DAE）。
编译器要做的是把指令/调度排好，让“搬运-计算”尽量重叠。

存储分配（Memory Allocation） 常见做法是寄存器/共享内存/缓存/全局内存分层放置，做 buffer 复用、生命周期分析、内存池、对齐与布局优化，减少峰值内存和访问开销

图里这两句话在问同一个核心问题：多级缓存下，tiling 怎么分层映射。
答案是：做分层 tiling（hierarchical tiling），每一级 tile 对应一层存储。

大 tile：放 L2 / HBM 友好的块（提高全局带宽利用）
中 tile：放 L1 / shared memory（提高复用）
小 tile：放寄存器 / 向量寄存器（提高算力利用）
所以常见会有 block tile -> warp/thread tile -> register tile（GPU）或 L2 tile -> L1 tile -> register tile（CPU/NPU）。

第二句“人工优化难不高”其实是：难度很高。原因：

张量布局复杂（NCHW/NHWC/blocked）
约束多（cache 容量、带宽、bank conflict、寄存器压力、并行映射）
参数耦合（tile size、unroll、vectorize、pipeline 相互影响）
因此 AI 编译器通常用：

规则模板（经验初值）
代价模型
autotuning/search
来自动找多级 tile 参数，而不是纯手工调。

传统编译器
以指令级/循环级为主。
关注 cache、分支、寄存器压力、ILP。
粒度通常是函数/基本块内局部优化。

AI 编译器
以张量算子和图级执行为主。
还要显式管多级存储（HBM/DDR/片上 SRAM/寄存器）、DMA、异步 pipeline、kernel 启动开销。
粒度常跨算子、跨 kernel，要做全图内存规划和调度。
分类：

循环优化（Loop Optimization）：循环展开、循环分块、循环重排、循环融合、循环拆分
指令优化（Instructions Optimization）：向量化、张量化  
存储优化（Memory Optimization）：访存延迟、存储分配

## 相比传统
• 更低实验开销：1）聚焦算子或者 kernel 级别的优化，而非整个程序。2）Cost Model 在 CPU 上模拟 NPU 执行，训练和推理推理的模拟速度要求足够快。
• 特定领域结构：针对神经网络算子或者 kernel 级别的优化，1）主要是高度循环化、张量化、并行化特点进行优化；2）大量相类似的算子计算模式。


• 如何可以让机器匹配手写优化性能？

建立一个足够大的搜索空间，保证可能的人工手写优化全部包含在这个搜索空间里面
快速地搜索这个空间，获取优化的实现

• 参数化 Parameterization：对调度优化问题进行建模，参数化优化空间一般由可参数化变换（Loop）的可能参数取值组合构成，因此需要调度原语进行参数化表示。Halide 将算法和调度解耦，TVM 提供调度模板。

• 成本模型 Cost Model：用来评价某一参数化下的调度性能，根据对调度额评价来指导最搜索到最优的调度策略。可以以运行时间、内存占用、编译后指令数来评价。实现方式主要有1）基于NPU硬件的黑盒模型；2）基于模拟的预定义模型；3）ML-Base 模型，通过机器学习模型来对调度性能进行预测。

• 搜索算法 Search Algorithm：确定初始化和搜索空间后，在搜索空间找找到达到性能最优的参数配置。常用的搜索算法有1）遗传算法、2）模拟退火算法、3）强化学习等。
